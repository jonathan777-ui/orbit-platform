# Alyxir — Website Generation Specs (per vertical & sub-niche)

The design brief the merge engine follows so every generated site has a distinct, intentional
look — not one template recolored. Each spec pairs with the vertical's KB (positioning §1,
website copy §12, FAQ §7, compliance §5). **LIVE** = wired into the generator's theme today;
**SPEC** = ready to wire when the full 20-vertical taxonomy lands.

Format per vertical: **Aesthetic** · **Palette** (accent / accent-2 / hero base) ·
**Type** (display / body) · **Hero** · **Emphasize** · **Imagery & icons** · **Mood** · **Note**.

## How the engine uses a spec
1. Pick the vertical (and sub-niche) → load its **theme** (palette, type pairing, hero variant) and its **KB** (positioning, website copy, FAQ, compliance).
2. Render the matching **hero treatment** (industrial / elegant / luxe / editorial / fresh / clean) — distinct type + layout, not a recolor.
3. Pull section copy from KB §12, service cards from the niche tree §14, agent answers from FAQ §7 + quick-replies §13, and hard limits from compliance §5.
4. Apply bilingual EN/ES from the KB's paired copy; mirror the dialect/register from §2 + the shared language layer.

Add a new vertical by: writing its KB file (template §0–§14) **and** one spec block here. The theme map + KB give the generator everything it needs to ship an eye-popping, accurate, compliant site.
